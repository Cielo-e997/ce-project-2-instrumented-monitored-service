# Incident Analysis

## Scenario

During testing, simulated application activity generated logs and metrics that were analyzed through CloudWatch.

By observing the monitoring dashboard and inspecting log events, it was possible to track how requests were processed by the system.

---

## Investigation Process

When investigating system behavior, both logs and metrics were examined.

Metrics provided a high-level view of system performance, while logs provided detailed information about individual requests.

Using both sources of information together made it possible to understand what the system was doing internally.

---

## Findings

The monitoring tools provided useful visibility into the application.

Logs allowed request processing steps to be traced, while metrics allowed patterns such as latency changes to be identified.

---

## Lessons Learned

This process demonstrated how observability tools help diagnose system behavior.

Instead of relying on guesswork, logs and metrics provide real operational data that can guide troubleshooting.
