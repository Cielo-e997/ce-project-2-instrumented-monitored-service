# Instrumentation

## Logging Strategy

The application uses structured logging to provide detailed information about system behavior.

Instead of plain text logs, the application produces log entries in JSON format.  
Each log entry contains fields such as:

- timestamp
- log level
- event name
- request path
- latency
- correlation ID

Structured logging makes it easier to search and analyze logs using CloudWatch.

---

## Correlation IDs

Each request processed by the application is assigned a unique correlation ID.

This identifier allows logs generated during the lifecycle of a request to be connected together.

Correlation IDs make it easier to trace the execution path of a request and diagnose issues.

---

## Custom Metrics

The application publishes custom metrics to CloudWatch to track system performance.

Examples of metrics generated by the application include:

- request count
- request latency
- error occurrences

These metrics allow system behavior to be monitored over time.

---

## Why Instrumentation Matters

Instrumentation provides visibility into the system while it is running.

Logs provide detailed information about individual events, while metrics allow trends and patterns to be observed.

Together they create the foundation for effective monitoring and troubleshooting.
